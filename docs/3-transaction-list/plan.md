# transaction-list

> 流水管理：用户可以查看、筛选、编辑和删除已记录的收支流水

## 目标

1. 用户可以按时间倒序查看所有流水记录
2. 用户可以通过时间范围和类型筛选流水
3. 用户可以查看流水明细并进行编辑或删除
4. 流水按日分组展示，清晰呈现每日收支情况

## 范围

**包含:**

- 流水列表（按时间倒序，按日分组）
- 时间范围筛选
- 类型筛选（支出/收入/转账）
- 流水明细页（查看完整信息）
- 编辑流水（修改金额、分类、备注、时间等）
- 删除流水

**不包含:**

- 按分类筛选
- 按账户筛选
- 多账本管理
- 批量操作（批量删除/编辑）
- 搜索功能
- 导出功能

## 核心用户场景

- 场景 A: 用户想查看今天的消费记录，打开流水页面，看到按日分组的流水列表，今天的记录在最上方
- 场景 B: 用户想查看上个月的所有支出，通过时间范围筛选选择上月，再通过类型筛选选择"支出"
- 场景 C: 用户发现一笔记录的金额输入错了，点击该记录进入明细页，修改金额后保存
- 场景 D: 用户发现一笔重复记录，点击进入明细页，选择删除

## 背景

流水列表是用户查看和管理已记录数据的核心页面。用户在记账后需要能方便地回顾、核对、修正记录。MVP 阶段提供基础的查看、筛选（时间+类型）和编辑删除能力，满足日常使用需求。
