# account-system

> 用户账号系统：提供注册、登录、退出功能，保障用户数据安全隔离

## 目标

1. 用户可以通过账号密码注册新账户
2. 用户可以通过账号密码登录系统
3. 用户可以安全退出登录
4. 系统通过 Token 鉴权保护用户数据

## 范围

**包含:**

- 账号密码注册
- 账号密码登录
- Token 鉴权与刷新机制
- 退出登录

**不包含:**

- 找回密码
- 邮件/短信验证
- 第三方登录（微信、Google 等）
- 用户头像/昵称等个人资料编辑

## 核心用户场景

- 场景 A: 新用户首次使用，通过注册页面输入账号密码完成注册，自动登录进入首页
- 场景 B: 已注册用户打开应用，输入账号密码登录，进入首页开始记账
- 场景 C: 用户在"我的"页面点击退出登录，返回登录页面
- 场景 D: 用户 Token 过期后，系统自动刷新 Token，用户无感知继续使用

## 背景

账号系统是所有功能的前置依赖。用户必须先注册登录才能使用记账、查看报表等核心功能。MVP 阶段采用最简方案：账号密码注册登录，不涉及邮件、短信等外部服务。

## 验收清单

- [ ] 用户可以通过账号密码成功注册，注册后自动登录
- [ ] 已注册用户可以通过账号密码成功登录
- [ ] 登录失败时显示明确的错误提示（账号不存在/密码错误）
- [ ] 登录后 Token 鉴权正常工作，访问受保护资源不报错
- [ ] Token 过期后可自动刷新，用户无需重新登录
- [ ] 用户可以成功退出登录，退出后需重新登录才能访问
