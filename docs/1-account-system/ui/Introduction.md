# Keep Account -- 前端设计说明

> 本文档为前端工程师提供设计实现指导，涵盖所有需求模块的统一视觉方向。

## 设计理念

**温暖极简 (Warm Minimalism)**

记账是日常高频操作，设计上追求：
- **零摩擦录入**: 记账页以金额输入为中心，所有辅助信息退到次要位置
- **清晰的数据层级**: 金额用大号字，分类用图标，时间和备注用小字
- **情感化的色彩**: 收入=绿色（正向激励），支出=红色（提醒注意），不是冰冷的纯数据展示
- **安静的界面**: 大量留白，轻阴影，让用户专注内容

## 响应式策略

三端布局有明显差异，各自体现平台交互特点，不是简单地把手机端放大。

### 断点与导航

| 断点 | Tailwind 前缀 | 导航方式 | 说明 |
|------|---------------|----------|------|
| < 768px (手机) | 默认 | 底部 Tab Bar (紧凑) | 全宽, pb-8 安全区 |
| 768px-1023px (平板) | `md:` | 底部 Tab Bar (宽松) | 加大触控区域, pb-4, Tab 居中 max-w-lg |
| >= 1024px (桌面) | `lg:` | 左侧侧边栏 (w-56/224px) | 带 Logo+应用名, 文字导航项, 底部用户信息 |

### 各页面桌面端布局要点

桌面端充分利用宽屏空间，采用多栏布局：

| 页面 | 桌面端布局 |
|------|-----------|
| 记账页 | **双栏**: 左栏记账表单(卡片样式) + 右栏"最近记录"列表(w-80~w-96) |
| 流水页 | **主从双栏**: 左栏流水列表(max-w-2xl) + 右栏详情面板(sticky, w-80~w-96) |
| 报表页 | **图表并排**: 总览3列卡片 + 下方环形图与趋势图双列并排(lg:grid-cols-2) |
| 我的页 | 侧边栏 + 居中内容区(w-[420px]) |
| 登录页 | 居中表单卡片, 无侧边栏 |

### 桌面端特有元素

- 顶部标题栏: `<header>` 显示页面标题 + 副标题, 仅桌面端可见 (`hidden lg:flex`)
- 表单卡片化: 桌面端的表单区域加上白色背景+圆角+阴影+边框，形成独立卡片
- 详情面板: 流水页桌面端用右侧固定面板替代手机端的 Bottom Sheet 弹窗
- 选中高亮: 流水列表选中项左侧蓝色边框 (`lg:border-l-2 lg:border-l-blue-500`)

## 导航结构

### 底部 Tab Bar (手机/平板)

4 个 Tab，从左到右：
1. **记账** (首页) -- 路由 `/`
2. **流水** -- 路由 `/transactions`
3. **报表** -- 路由 `/reports`
4. **我的** -- 路由 `/me`

- 图标使用 Lucide Icons (24px)
- 激活态: 蓝色 (blue-600/blue-400)，未激活: 灰色 (gray-400/gray-500)
- Tab Bar 高度: 56px + safe-area-inset-bottom (iOS)
- Tab Bar 背景: white/gray-800，上边框: gray-200/gray-700

### 侧边栏 (桌面)

- 宽度: 224px (w-56)，纵向排列
- 顶部: Logo 图标 (40x40px) + 应用名 "记账本" + 副标题 "Keep Account"
- 导航项: 图标 (20px) + 文字标签，水平排列，圆角高亮背景
- 激活态: 蓝色文字 + 蓝色浅背景 (blue-50/blue-900)
- 底部: 用户头像 + 用户名 (分割线隔开)
- 背景: white/gray-800，右边框: gray-200/gray-700

## 各页面设计要点

### 1. 登录页 / 注册页

**布局**: 垂直居中，最大宽度 400px
- Logo: 64x64px 蓝色圆角方块 + 货币符号 SVG
- 标题: "记账本"，副标题: "轻松记录每一笔"
- 表单: 账号 + 密码输入框，高度 44px
- 主按钮: 全宽蓝色，高度 48px
- 注册页增加: 返回箭头 + 确认密码输入框

**状态**: 正常态、错误态（输入框红色边框 + 错误消息）、加载态（按钮 loading）

**注意**: 登录/注册页不显示 Tab 导航

### 2. 记账页 (核心)

**布局**: 从上到下
1. 类型切换: 支出/收入/转账 -- 使用 Segmented Control 样式
2. 金额输入: 48px 大号数字输入框，居中，左侧人民币符号
3. 分类选择: 5 列网格，每项 = 圆形图标 + 文字标签
4. 备注 + 日期: 一行排列，备注占大部分宽度
5. 保存按钮: 全宽，固定在底部

**金额输入**:
- 使用系统原生键盘: `<input type="number" inputmode="decimal">`
- 移动端会自动弹出系统数字键盘，桌面端使用普通输入
- 不做自定义数字键盘，保证多端（Web/iOS/Android/Desktop）行为一致
- 隐藏 number input 默认的上下箭头: `appearance: textfield`

**分类颜色编码**:
- 选中分类: 蓝色背景 (blue-50) + 蓝色边框 (border-2 border-blue-500)
- 收入模式选中: 绿色背景 (green-50) + 绿色边框

**保存按钮**:
- 全宽圆角按钮，固定在底部（Tab Bar 上方）
- 支出模式: 蓝色 (blue-600)
- 收入模式: 绿色 (green-600)

### 3. 流水页

**布局**:
1. 页面标题 "流水"
2. 筛选栏: 类型 Tab (全部/支出/收入/转账) + 日期范围按钮
3. 按日分组列表

**每日分组**:
- 日期标题: "今天" / "昨天" / "2月26日 周四"
- 右侧小计: "支出 xx / 收入 xx"
- 流水项: 白色卡片，圆角 12px

**流水项结构**:
```
[类型图标(36px圆)] [分类名 + 备注(灰色小字)] ---- [金额(右对齐) + 时间(灰色小字)]
```

**金额颜色**: 支出 = 红色 (-38.50), 收入 = 绿色 (+5,000.00), 转账 = 灰色

**明细页**:
- 手机/平板: 底部弹出模态框（Bottom Sheet），包含完整信息 + 编辑/删除按钮
- 桌面端: 右侧固定详情面板 (sticky)，点击列表项直接在右栏展示详情，不弹窗

### 4. 报表页

**布局**: 可滚动
1. 页面标题 "报表"
2. 月份切换: 左箭头 + "2026年2月" + 右箭头
3. 总览卡片: 3 列等宽 (收入/支出/结余)
4. 分类占比: 环形图 + 分类列表
5. 趋势图: 柱状图

**总览卡片配色**: 收入=绿色数字，支出=红色数字，结余=蓝色数字

**分类占比图**:
- 环形图直径: 手机 180px / 桌面 220px，内圈 110px / 140px (显示总支出)
- 颜色序列: blue-600, amber-500, emerald-500, purple-500, red-500, pink-500, gray-400
- 下方分类列表: 色块圆点 + 分类名 + 金额 + 百分比 + 右箭头(可下钻)

**趋势柱状图**:
- 柱子用蓝色渐变 (blue-200 到 blue-600)
- x 轴: 日期标签 (桌面端显示28天完整数据，手机端可简化)
- 图表高度: 手机 h-40 / 桌面 h-56
- 底部图例

**桌面端布局**: 环形图+分类列表 与 趋势柱状图 **并排双列** (`lg:grid-cols-2`)，充分利用宽屏

### 5. 我的页面

**布局**: 列表式菜单
1. 用户信息卡片 (头像圆圈 + 用户名 + 状态)
2. 菜单项: 分类管理、主题设置
3. 退出登录按钮 (红色文字，独立卡片)

## 深色模式实现指南

使用 Tailwind `dark:` 前缀，通过 `<html class="dark">` 切换。

**核心替换规则**:
| 浅色 | 深色 | 用途 |
|------|------|------|
| bg-gray-50 | dark:bg-gray-900 | 页面背景 |
| bg-white | dark:bg-gray-800 | 卡片/表面 |
| border-gray-200 | dark:border-gray-700 | 边框 |
| text-gray-900 | dark:text-gray-50 | 主文字 |
| text-gray-500 | dark:text-gray-400 | 次要文字 |
| shadow-sm | dark:border dark:border-gray-700 | 深色模式用边框替代阴影 |

## 图标使用

使用 **Lucide React** (lucide-react) 图标库:

```bash
npm install lucide-react
```

常用图标映射:
- 导航: `Clock` (记账), `List` (流水), `PieChart` (报表), `User` (我的)
- 分类(支出): `Coffee` (餐饮), `ShoppingBag` (购物), `Car` (交通), `Gamepad2` (娱乐), `Heart` (医疗), `GraduationCap` (教育), `Home` (住房), `Smartphone` (通讯), `MoreHorizontal` (其他)
- 分类(收入): `Briefcase` (工资), `Gift` (红包), `TrendingUp` (投资), `DollarSign` (兼职), `MoreHorizontal` (其他)
- 操作: `ChevronLeft`, `ChevronRight`, `Trash2`, `Pencil`, `X`, `Check`, `Delete` (退格)

## 动画建议

| 场景 | 动画 | 时长 |
|------|------|------|
| 按钮点击 | scale(0.97) + 颜色变化 | 100ms |
| Tab 切换 | 背景色过渡 | 200ms |
| 模态框弹出 | 从底部滑入 + 背景淡入 | 200ms ease-out |
| Toast 提示 | 从顶部滑入 + 停留 + 滑出 | 200ms + 2s + 200ms |
| 页面数据加载 | 骨架屏闪烁 | 1.5s pulse |

**注意**: 始终检查 `prefers-reduced-motion`，有此设置时跳过动画。
