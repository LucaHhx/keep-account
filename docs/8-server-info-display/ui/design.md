# 服务端点展示 -- UI 设计规范

> 需求: server-info-display | 角色: ui

## 设计目标

在"我的"页面和登录页面以**低调、辅助性**的方式展示当前 App 连接的服务端点，帮助用户/开发者快速确认请求目标，不干扰主要功能操作。

## 设计原则

1. **低调辅助**: 端点信息是辅助信息，不应抢占视觉焦点
2. **一致融合**: 与现有页面风格无缝融合，看起来像原生一部分
3. **可读性**: 尽管字号小，但要保证足够的对比度和清晰度
4. **防溢出**: 端点地址可能很长，需要优雅处理文本溢出

## 视觉规范

### 端点文本样式

| 属性 | 浅色模式 | 深色模式 |
|------|----------|----------|
| 字号 | 12px (text-xs) | 12px (text-xs) |
| 字重 | 400 (normal) | 400 (normal) |
| 颜色 | `#9CA3AF` (gray-400) | `#6B7280` (gray-500) |
| 行高 | 1.5 | 1.5 |
| 字体 | Inter, monospace 风格数字 | 同左 |

> 使用 Text Muted 层级颜色，确保信息存在但不抢眼

### 文本溢出处理

- 使用 `truncate` (overflow: hidden; text-overflow: ellipsis; white-space: nowrap)
- 端点文本随容器宽度自适应，不换行

## 页面设计详情

### "我的"页面 -- 用户信息卡片

**位置**: 在"已登录"状态文本下方，作为第三行辅助文本

**布局结构**:
```
[头像图标]  用户名
            已登录
            http://192.168.1.100:5723
```

**Tailwind 类名**:
```
text-xs text-gray-400 dark:text-gray-500 truncate
```

**间距**: 无额外 margin-top，紧随"已登录"文本（由 min-w-0 容器的默认行距分隔）

### 登录页面

**位置**: 在副标题"轻松记录每一笔"下方

**布局结构**:
```
[Logo 图标]
记账本
轻松记录每一笔
http://192.168.1.100:5723
```

**Tailwind 类名**:
```
text-xs text-gray-400 dark:text-gray-500 mt-1
```

**间距**: `mt-1` (4px)，与副标题保持紧凑关系

## 响应式表现

端点文本在所有断点下使用相同样式（text-xs + text-gray-400），无需针对不同断点做差异化处理。

| 断点 | 表现 |
|------|------|
| 手机 (< 768px) | 文本可能 truncate，显示大部分 URL |
| 平板 (768px+) | 容器更宽，通常能完整显示 |
| 桌面 (1024px+) | 容器有 max-width 限制，显示完整或接近完整 |

## 深色模式

使用 `dark:text-gray-500` 替代浅色模式的 `text-gray-400`，遵循设计系统中 Text Muted 的深浅模式对应关系。在深色背景上保持足够低调但可辨认。

## 无障碍

- 端点文本虽然字号小，但对比度满足 WCAG AA 标准（gray-400 on white = 4.56:1）
- 文本可被选中复制，方便开发者使用
