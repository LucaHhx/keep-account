# server-info-display

> 服务端点展示：在"我的"页面和登录页面显示当前 App 连接的服务端点，方便用户/开发者查看请求目标

## 目标

1. 用户在"我的"页面可以看到当前 App 连接的服务端点
2. 用户在登录页面可以看到当前 App 连接的服务端点
3. 帮助用户/开发者快速确认 App 当前请求的后端服务端点

## 范围

**包含:**

- 在"我的"页面用户信息区域展示当前连接的服务端点
- 在登录页面展示当前连接的服务端点
- 端点地址可能是 IP 地址（如 `192.168.1.100:5723`）也可能是域名（如 `api.example.com`）

**不包含:**

- 服务器切换功能
- 服务器健康检查/状态检测
- 自定义服务端点配置

## 核心用户场景

- 场景 A: 用户打开"我的"页面，在用户信息卡片中看到当前连接的服务端点（如 `http://192.168.1.100:5723` 或 `https://api.example.com`），确认请求目标正确
- 场景 B: 用户在登录页面看到当前连接的服务端点，登录前就能确认目标服务器
- 场景 C: 开发者在调试时切换不同环境（dev/staging/prod），通过登录页或"我的"页面快速确认当前连接的是哪个端点

## 背景

多端应用在不同环境（开发、测试、生产）下可能连接不同的后端服务端点。当前 App 没有直观的方式让用户或开发者确认当前连接的端点地址。在"我的"页面和登录页面展示这个信息可以方便排查问题、确认环境。端点地址可能是 IP（如 `192.168.1.100:5723`）也可能是域名（如 `api.example.com`）。
