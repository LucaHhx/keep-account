# cloud-sync

> 云同步：所有数据存储在云端，多端登录同账号自动同步

## 目标

1. 用户数据安全存储在云端
2. 用户在任意终端登录同一账号后可访问完整数据
3. 多端之间的数据保持一致

## 范围

**包含:**

- 所有数据（流水、分类等）通过 API 实时存取云端
- 多端登录同账号数据一致
- 数据通过 API 实时获取，保证最新

**不包含:**

- 离线记账 / 本地缓存
- 离线同步队列
- 冲突处理机制
- 端到端加密

## 核心用户场景

- 场景 A: 用户在手机上记了几笔账，回家后打开电脑，登录同一账号，看到手机上记的所有流水
- 场景 B: 用户在电脑上修改了一笔流水，切换到手机刷新后看到修改已生效
- 场景 C: 用户在没有网络的环境下打开应用，看到提示需要联网才能使用

## 背景

MVP 阶段采用最简的云同步方案：所有数据通过 API 实时读写云端，不做本地缓存和离线支持。这大幅降低了开发复杂度（无需处理离线队列、冲突合并等问题），同时满足了"多端数据一致"的核心需求。离线能力作为后续迭代方向保留。

PRD 中提到的离线缓存、同步队列、冲突处理策略等技术方案，已在 MVP 范围讨论中确认不做，供后续迭代参考。

## 验收清单

- [ ] 所有记账数据通过 API 存储在云端
- [ ] 同一账号在不同终端登录后可查看相同的数据
- [ ] 一端的操作（新增/编辑/删除）在其他端刷新后可见
- [ ] 无网络时应用给出明确的联网提示
