# multi-platform

> 多端编译与测试：建立完整的多平台构建流程，支持开发团队在桌面（macOS/Windows/Linux）、iOS、Android 平台上进行开发调试与构建发布

## 目标

1. 开发团队可以通过统一的脚本命令启动各平台的开发调试环境
2. 开发团队可以通过统一的脚本命令构建各平台的发布产物
3. 前端代码一套代码适配桌面端和移动端
4. 各平台构建产物可正常安装运行，功能与 Web 端一致

## 范围

**包含:**

- 桌面端（macOS/Windows/Linux）开发调试与构建脚本
- iOS 模拟器开发调试与构建脚本
- Android 模拟器/真机开发调试与构建脚本（生成 APK）
- 各平台构建产物的基本功能验证

**不包含:**

- 应用商店发布（App Store / Google Play）
- CI/CD 自动化流水线
- 代码签名与证书管理
- 多平台 UI 适配优化（响应式布局属于其他需求）
- 自动化测试框架搭建

## 核心用户场景

> 注: 本需求的"用户"是开发团队成员，非终端用户

- 场景 A: 开发者修改前端代码后，运行桌面开发命令，在桌面窗口中实时预览变更并调试
- 场景 B: 开发者需要验证移动端效果，运行 iOS 开发命令，在 iOS 模拟器中查看应用并调试
- 场景 C: 开发者需要在 Android 设备上测试，运行 Android 开发命令，在模拟器或真机中调试
- 场景 D: 开发者完成一个版本开发，分别运行各平台构建命令，生成对应的安装包/产物
- 场景 E: 新加入的开发者通过查看项目脚本列表，快速了解项目支持哪些平台以及对应的开发/构建命令

## 背景

记账本是一个面向多端的应用（桌面 + iOS + Android）。当前项目仅有前端 Web 开发脚本，缺少多端开发与构建命令。开发团队需要完整的多平台构建脚本来支撑日常开发调试和版本构建。

## 验收清单

- [ ] 项目中包含覆盖所有目标平台的开发调试脚本（桌面、iOS、Android）
- [ ] 项目中包含覆盖所有目标平台的构建脚本（桌面、iOS、Android）
- [ ] 桌面开发命令可启动桌面窗口，前端热更新正常工作
- [ ] 桌面构建命令可生成桌面安装包
- [ ] iOS 开发命令可在模拟器中启动应用并调试（需 macOS + Xcode 环境）
- [ ] iOS 构建命令可生成 iOS 构建产物（需 macOS + Xcode 环境）
- [ ] Android 开发命令可在模拟器或真机中启动应用并调试
- [ ] Android 构建命令可生成 APK 安装包
- [ ] 各平台应用启动后可正常显示前端页面，基本交互无异常
