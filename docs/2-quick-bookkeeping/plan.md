# quick-bookkeeping

> 3秒记账核心功能：用户通过极简操作路径快速完成一笔支出、收入或转账记录

## 目标

1. 用户可以在3次操作内完成一笔记账（输入金额 + 选分类 + 保存）
2. 支持支出、收入、转账三种记账类型
3. 通过默认值策略减少用户输入（时间默认当前、分类智能推荐）
4. 系统提供预设分类，用户可自行添加新分类

## 范围

**包含:**

- 新增支出记录
- 新增收入记录
- 新增转账记录
- 金额输入（系统原生键盘，inputmode=decimal）
- 分类选择（常用/最近优先展示）
- 备注输入（可选）
- 时间默认为当前时间（可修改）
- 系统预设分类管理
- 用户自定义添加分类（不可删除预设分类）

**不包含:**

- 保存后撤销
- 多账户选择（每用户单账户）
- 币种选择（固定人民币）
- 语音/拍照记账
- 重复/定期记账

## 核心用户场景

- 场景 A: 用户刚消费完，打开应用进入首页，输入金额 38.5，点选"餐饮"分类，点保存，完成记账
- 场景 B: 用户收到一笔工资，切换到"收入"类型，输入金额，选择"工资"分类，保存
- 场景 C: 用户从银行卡取现金，切换到"转账"类型，输入金额，保存
- 场景 D: 用户发现预设分类中没有"宠物"，进入分类管理添加自定义分类"宠物"，之后记账时可选择

## 背景

3秒记账是产品的核心差异化价值。记账软件最大的使用障碍是"麻烦"，通过极简的操作路径（金额+分类+保存）和智能默认值，让用户几乎无摩擦地完成记账，从而提高使用频率和留存。

- [ ] 时间默认为当前时间，用户可手动修改
