本需求遵循统一设计系统，详见 `docs/1-account-system/ui/design.md`。

## 页面设计

### 记账页 (BookkeepingPage)

本应用的核心页面，也是首页 (Tab 1)。

**设计目标**: 3次操作完成记账（输入金额 + 选分类 + 保存）

**布局结构** (从上到下):
1. **类型切换栏**: Segmented Control 样式 (支出/收入/转账)
2. **金额输入区**: 48px 大号数字输入框 + 人民币符号，使用系统原生键盘
3. **分类选择区**: 5列网格，圆形图标 + 文字，选中态高亮
4. **附加信息栏**: 备注输入 + 日期选择，一行排列
5. **保存按钮**: 全宽，固定底部

**关键设计决策**:
- 使用 `<input type="number" inputmode="decimal">` 调用系统键盘，不做自定义数字键盘，避免多端行为不一致
- 支出模式: 保存按钮蓝色 (blue-600)，分类选中蓝色边框
- 收入模式: 保存按钮绿色 (green-600)，分类选中绿色边框
- 转账模式: 隐藏分类区域，保存按钮灰色 (gray-600)，金额灰色 (gray-700)
- 金额显示颜色跟随类型 (支出=深色, 收入=绿色, 转账=灰色)

### 分类管理页 (CategoryManagePage)

从"我的"页面进入。

**布局**:
- 顶部导航: 返回箭头 + 标题"分类管理"
- 类型切换: 支出/收入 Segmented Control
- 分类列表: 竖向列表展示，每项包含图标 + 名称
- 预设分类: 右侧显示"预设"标签，无删除按钮
- 自定义分类: 右侧有删除按钮 (红色垃圾桶图标)
- 底部: "添加自定义分类" 按钮 (虚线边框 + 加号图标)

**添加分类弹窗** (Bottom Sheet):
- 拖拽指示条 + 关闭按钮
- 表单: 分类名称输入框 + 分类类型选择 + 图标选择 (6列网格)
- 提交按钮: "添加分类" 蓝色全宽按钮
