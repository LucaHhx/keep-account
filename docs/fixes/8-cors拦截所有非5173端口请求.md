# FIX-8: CORS 拦截所有非 5173 端口请求 + iOS POST body 丢失

> 日期: 2026-03-02 | 状态: 已修复 | 严重程度: P0

## 现象

1. **网页版 403**: Vite dev server 运行在 5175 端口（非默认 5173），所有 API 请求被后端 CORS 中间件拦截返回 403 Forbidden
2. **iOS 版 EOF**: WKWebView 丢弃 POST 请求体，后端收到空 body 返回"参数校验失败: EOF"
3. **Tauri 桌面版请求失败**: tauriAdapter 使用相对 URL `/api/v1`，原生 HTTP 不支持相对路径

## 根因分析

### CORS 403
- `cors.go` 的 `AllowOriginFunc` 硬编码 `http://localhost:5173`
- Vite 端口不是 5173 时（被占用自动递增），origin 不匹配 → 403
- `isPrivateIPOrigin` 中 `net.ParseIP("localhost")` 返回 nil，localhost 不被识别

### iOS POST body 丢失
- Tauri v2 在 iOS 上使用 WKWebView，其 `WKURLSchemeHandler` 存在 Apple bug：POST body 被丢弃
- Axios 默认使用 XMLHttpRequest，请求经 WKWebView 网络层后 body 为空

### Tauri 桌面 baseURL 问题
- `__TAURI_DEV_HOST__` 在桌面 dev 模式为空，baseURL 降级为 `/api/v1`（相对路径）
- tauriAdapter 使用 NSURLSession 原生 HTTP，不支持相对路径 → 请求失败

## 修复方案

### 1. CORS（后端）
dev 模式下允许所有 `http://localhost:*` origin，无论端口：
```go
if config.C.Server.IsDev() {
    u, err := url.Parse(origin)
    if err == nil && u.Hostname() == "localhost" {
        return true
    }
}
```
同时修复 `isPrivateIPOrigin` 对 "localhost" 的处理。

### 2. iOS POST body（前端 + Tauri 插件）
- 引入 `@tauri-apps/plugin-http` 插件，通过原生网络栈发请求
- 自定义 Axios `tauriAdapter`，Tauri 环境下自动使用原生 HTTP

### 3. baseURL 统一（前端）
Tauri 环境一律使用绝对 URL：
```typescript
const baseURL = isTauri
  ? `http://${(isDev && __TAURI_DEV_HOST__) || 'localhost'}:5723/api/v1`
  : '/api/v1';
```
