# FIX-9: 多端请求后端失败及网页跨域

> 日期: 2026-03-02 | 状态: 已修复 | 严重程度: P1

## 现象

1. macOS 客户端和 iOS 客户端无法请求到后端
2. 网页版产生跨域请求到后端

## 根因分析

1. **iOS dev 模式**：`.env.development` 中 `VITE_API_BASE_URL` 为空，baseURL 回退为 `http://localhost:5723/api/v1`，但 iOS 设备上的 localhost 指向设备自身而非运行后端的 Mac
2. **macOS/iOS 生产构建**：`.env.production` 硬编码局域网 IP `192.168.0.145:5723`，但 `.env.development` 未配置，导致 dev 模式下 macOS Tauri 可能使用 localhost 而非后端实际 IP
3. **网页版跨域**：前端 dev server (5173) 直接请求后端 (5723)，端口不同产生跨域；后端 CORS 已放行所有 origin，功能可用但不够规范
4. **`__TAURI_DEV_HOST__` 逻辑缺失**：`vite.config.ts` 定义了 `__TAURI_DEV_HOST__` 全局变量，但 `axios.ts` 未使用，iOS dev 模式无法获取宿主机 IP

## 修复方案

1. **统一 dev/production 后端地址**：在 `.env.development` 中设置 `VITE_API_BASE_URL=http://192.168.0.145:5723`，所有端在 dev 模式下直连后端
2. **恢复 `__TAURI_DEV_HOST__` 逻辑**：在 `axios.ts` 中使用 `__TAURI_DEV_HOST__` 作为 Tauri dev 模式的兜底，确保 iOS 设备能通过宿主机 IP 访问后端
3. **所有端直连后端**：不使用 Vite proxy，依赖后端 CORS 中间件（AllowOriginFunc 返回 true）处理跨域

## 变更文件

| 文件 | 修改说明 |
|------|----------|
| `web/src/lib/axios.ts` | 重构 `getBaseURL()` 函数：优先用 `VITE_API_BASE_URL`，Tauri 兜底用 `__TAURI_DEV_HOST__`，Web 兜底用 localhost |
| `web/.env.development` | 设置 `VITE_API_BASE_URL=http://192.168.0.145:5723` |
